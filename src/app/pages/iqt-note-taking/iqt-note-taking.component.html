UX Development for Security Visualization

Creating a user-centric notetaking feature for CRViz

UX Designer + Software Engineer

Framing the Problem

After cyber operators have loaded their data into CRViz, they find interesting patterns or features within the data. Needing to communicate these insights to outside users is important. People who aren't familiar with the data or even the underlying problem space your data exists within is common and it was a need to be addressed in CRViz

CHALLENGE

Security operators need to document insights

By annotating the visualization, network controllers retain insights gathered from the dataset

SOLUTION

Notetaking feature

A notetaking feature that's embedded in the visualization and has the ability to have notes imported into and exported from the visualization

SCOPE


UX Design

UX Engineering

Overview

CRViz is a browser-based visualization tool that uses JSON and an interactive enclosure diagram to visualize networks. Built with React, Redux, and d3, it uses circle- packing techniques to easily visualize over 10k nodes.

Notetaking as a feature isn't a new concept so I dove into researching how other products’ note-taking features function in an effort to incorporate sensible features from those into CRviz.

By understanding how users interact with those products, we can gain insight on required functionalities in our note-taking implementation. Identifying those features allows us to provide a user-centric experience that we can develop within our technical feasibility.

Existing implementation

Here's CRViz before the notetaking feature was implemented. Using d3, we presented the information of each node with a mouseover event. This limited us to only being able to use d3 elements to interact with our visualization.

We took the notetaking feature as an opportunity to abstract some d3 implementations into React giving us the flexibility to create a user-centric notes component.

Comparative Products

Google Keep

Google Keep provides a “post it” UI of note-taking with ability to create lists, draw on your notes, and create notes with images.

Evernote

Evernote is application designed for note taking, organizing, task lists, and archiving. It’s centered around the idea of notes being gathered within notebooks.

Sketches

Before anything was put into software, I drew up a couple views of the panel on the left-hand side for CRViz. I decided that listing the notes with the other controls for the visualization made the most sense for the direction of the notetaking feature.

Low-Fi Prototyping

Interactive Mockups

Through the design of this feature, I went through a couple iterations of how I wanted the notes to be listed so that the user could find them easier after taking a note. I came to the conclusion that having the notes on the left-hand side with interactions such as 1) hover and your note shows up on the visualization and 2) click and you zoom in onto your note, would have the desired effect in keeping with the projects other conventions.

First implementation

Our new note-taking feature moved a lot of functionality from d3 into React. This required a decent amount of re-engineering and plumbing via Redux.

Key Takeaways

Prototyping in low-fidelity is key

I worked quite a bit in Balsamiq and React and was happy that I decided to really hammer out details in Balsamiq before moving it to React. Prototyping interactions in a flexible data visualization tool like this can be challenging because there isn’t a “typical” workflow you can realistically expect from your user.

Get caught up in the weeds

I was always trying to push through the prototyping phase so that I could get something that people could see. Slow down! I really had to learn how to think about my work in the long term and realize that what I was designing in my product needed to fit the previously defined conventions. Getting stuck and critically thinking about what to do next was important and something my product manager wanted me to learn.
